## 📝 커밋 규칙

### 커밋 메시지 형식
```
[태그] 변경 내용 요약
```

### 태그 종류
| 태그 | 설명 |
|------|------|
| `[Add]` | 새로운 기능, 파일 추가 |
| `[Fix]` | 버그 수정 |
| `[Modify]` | 기존 기능 수정 |
| `[Remove]` | 파일, 기능 삭제 |
| `[Refactor]` | 코드 리팩토링 |
| `[Setting]` | 프로젝트 설정 변경 |

### 예시
```
[Add] 플레이어 이동 스크립트 추가
[Fix] 총알 충돌 오류 수정
[Setting] Physics2D 레이어 설정 변경
```

## 🔄 작업 시작 전 필수 사항

1. **항상 `git pull` 먼저 받기**
2. 본인 작업 브랜치로 체크아웃 후 작업
3. 작업 전 팀원과 담당 영역 공유 (충돌 방지)

---

## ⚠️ 충돌 방지 규칙

- **씬(Scene) 파일**은 동시 편집 금지 → 씬 작업 시 팀원에게 먼저 공유
- **ProjectSettings** 폴더는 한 명이 대표로 관리
- `packages-lock.json`은 Unity가 자동 생성하므로 임의 수정 금지
- 같은 스크립트 파일을 동시에 수정하지 않기

---

## 📁 폴더 구조 규칙

```
Assets/
├── 본인이름/         ← 개인 작업 폴더 (자유롭게 작업)
│   ├── Scenes/
│   ├── Scripts/
│   └── Prefabs/
├── Shared/           ← 공용 리소스 (수정 전 팀원과 협의)
│   ├── Scripts/
│   ├── Prefabs/
│   └── Sprites/
└── ...
```

---

## 🚫 Git에 올리지 않는 것들 (.gitignore)

- `Library/` 폴더
- `Temp/` 폴더
- `Logs/` 폴더
- `.DS_Store`, `Thumbs.db`
- 개인 IDE 설정 파일

---

## 💬 소통 규칙

- 작업 시작/완료 시 팀 채팅에 공유
- 씬, 공용 스크립트 수정 시 사전 공지
- 버그 발견 시 바로 팀원에게 공유
- 모르는 것은 혼자 해결하려 하지 말고 팀원과 상의

---

## 🛠️ Unity 협업 주의사항

- Unity 버전 통일 : **`(버전 기입)`**
- 씬 파일 저장 후 반드시 commit (미저장 상태로 push 금지)
- Prefab 수정 시 반드시 Apply 후 commit
- 물리, 레이어, 태그 추가 시 `ProjectSettings` 변경 사항을 팀원에게 공유
